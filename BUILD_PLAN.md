# ClipForge Build Plan

ClipForge — Technical Specification
AI-Powered Thumbnail Generator & Short-Form Clip Maker
For Lifestyle/Comedy YouTube Creators
1. Project Overview
ClipForge is a web application that takes a YouTube video URL and produces two outputs:
1. High-CTR Thumbnails — AI-generated thumbnail options using frames from the video, styled to match current trends in the lifestyle/comedy YouTube niche.
2. Short-Form Clip Suggestions — AI-detected "best moments" from the video, scored for virality, which the creator can preview, select, and export as vertical videos formatted for TikTok and YouTube Shorts.
Target user: A lifestyle/comedy YouTuber who posts semi-edited vlogs and comedy content (not podcast or talking-head format). The content has natural, unscripted energy — not heavily structured or dialogue-driven.
Key differentiator from OpusClip and similar tools: Those tools rely on generic audio-energy and keyword-density signals that work well for podcasts/interviews but fail for loosely structured lifestyle/comedy content. ClipForge uses Claude's language model to understand comedic timing, narrative arcs, and relatability — things that generic signal detection misses entirely.

4.2 Thumbnail Style Rules (Based on 2026 Trends)
Encode these as part of the Claude system prompt for thumbnail generation:
THUMBNAIL STYLE GUIDE — LIFESTYLE/COMEDY NICHE (2026):  CURRENT WINNING PATTERNS: - Simplicity is king. One focal point, minimal clutter. - Expressive close-up faces showing GENUINE emotion (surprise, confusion, excitement, disgust). - Text: 0-6 words maximum. Short intriguing questions or bold statements. - Clean sans-serif fonts, large size, high contrast against background. - Color: Bold, saturated palette OR desaturated/pastel with one pop color. - Authenticity over polish — slightly "raw" feeling outperforms overly designed.  WHAT TO AVOID: - Multiple text elements competing for attention. - Busy backgrounds with too many objects. - Generic stock-photo energy. - Outdated styles: multi-colored fills, excessive arrows, clickbait red circles. - Small text that's unreadable on mobile.  FORMAT RULES: - Text should NEVER obscure the subject's face. - If using text, it should be readable at phone-screen thumbnail size (~160x90px). - Leave breathing room — don't fill every pixel. - The thumbnail must make sense WITHOUT the title (they work as a pair, not a repeat).

5. Feature 2: Short-Form Clip Detection & Export
5.1 Pipeline
YouTube URL     │     ▼ [1] Download video (yt-dlp, best quality up to 1080p)     │     ▼ [2] Extract audio → Transcribe (faster-whisper)     ├── Word-level timestamps     ├── Speaker diarization (if multiple speakers)     └── Output: timestamped transcript JSON     │     ▼ [3] Audio/Video Signal Analysis     ├── Audio energy envelope (volume peaks = excitement/laughter)     ├── Speech rate changes (fast = energy, slow = emphasis)     ├── Silence gaps (scene transitions, comedic pauses)     ├── Scene change detection (visual transitions)     └── Output: signal_data JSON with timestamps     │     ▼ [4] AI Clip Detection (Claude API) — THE CORE LOGIC     ├── Input: full transcript + signal_data + trend context     ├── Claude identifies 5-15 potential clips     ├── Each clip gets a virality score + breakdown     └── Output: ranked clip list with metadata     │     ▼ [5] Frontend: Clip Selector Dashboard     ├── Preview each suggested clip (embedded video player with start/end)     ├── See virality score breakdown     ├── Adjust start/end times with trim handles     ├── Toggle caption styles     └── Select clips for export     │     ▼ [6] Export Engine (FFmpeg)     ├── Crop to 9:16 vertical with face tracking     ├── Burn in animated captions     ├── Trim to selected duration     ├── Encode for TikTok (H.264, AAC) and Shorts (same)     └── Output: downloadable .mp4 files 
5.2 Clip Detection: The Claude Scoring System
This is the most important part of the app — what makes it better than OpusClip.
System Prompt for Clip Detection
CLIP_DETECTION_SYSTEM_PROMPT = """ You are an expert short-form video strategist specializing in lifestyle/comedy content for TikTok and YouTube Shorts.  You will receive: 1. A full timestamped transcript of a YouTube video 2. Audio/visual signal data (energy peaks, scene changes, speech rate) 3. Current trend context for lifestyle/comedy short-form content  Your job: Identify the 5-15 best moments from this video that would perform well as standalone short-form clips (15-60 seconds each).  IMPORTANT: This creator makes lifestyle/comedy content that is NOT heavily edited or structured like a podcast. The humor comes from natural moments, reactions, relatable situations, and comedic timing — NOT from structured setups.  For each potential clip, return:  {   "clips": [     {       "start_time": "MM:SS",       "end_time": "MM:SS",       "duration_seconds": <int>,       "transcript_snippet": "<the key dialogue/text>",       "clip_title": "<suggested title/caption for posting>",       "hook_text": "<the first 2 seconds — what stops the scroll>",        "virality_score": <1-100>,       "score_breakdown": {         "hook_strength": <1-100>,         "standalone_clarity": <1-100>,         "emotional_arc": <1-100>,         "trend_alignment": <1-100>,         "rewatch_potential": <1-100>       },        "reasoning": "<2-3 sentences explaining why this clip would work>",       "suggested_caption": "<TikTok/Shorts caption with hashtags>",       "suggested_duration": "15s|30s|60s",       "clip_type": "punchline|reaction|relatable_moment|story_hook|visual_gag|hot_take|transformation"     }   ] }  SCORING RUBRIC:  hook_strength (30% weight): - Does the clip open with something that stops scrolling in 1-2 seconds? - Strong hooks: unexpected statement, relatable complaint, visual surprise, mid-action start - Weak hooks: slow buildup, context-dependent opening, generic greeting - For lifestyle/comedy: "I can't believe I just..." or showing a funny situation MID-moment scores high  standalone_clarity (25% weight): - Can someone who has NEVER seen this channel understand and enjoy this clip? - No inside jokes that require context - The situation/joke/moment is self-contained - Deduct points if the clip requires knowing what happened before  emotional_arc (20% weight): - Does the clip build to a payoff? - Comedy: setup → punchline or escalating absurdity → reaction - Lifestyle: situation → twist or relatable moment → genuine reaction - The payoff should land within the clip, not require watching more  trend_alignment (15% weight): - Does this match formats currently performing on TikTok/Shorts? - Current high-performing formats: relatable daily life moments, "watch this" challenge-style,   unexpected reactions, hot takes on everyday things, transformation/before-after reveals - Caption/text overlay hooks that match trending styles  rewatch_potential (10% weight): - Is there a subtle detail, double meaning, or quick moment worth catching again? - Facial expressions, background details, timing that rewards a second watch - Comedy clips with layered humor score highest here  CRITICAL RULES: - NEVER suggest a clip that starts with dead air, silence, or boring setup - Every clip MUST have a strong first 2 seconds - Prefer clips that END on a high note (laughter, reaction, punchline) rather than trailing off - If a moment is funny but needs 30 seconds of context, expand the clip to include minimal context - For comedy: timing matters more than topic. A well-timed reaction > a long funny story - Suggested durations: 15s for punchy moments, 30s for short stories, 60s for mini-narratives """

5.5 Caption Generation & Burn-In
Animated captions are critical for short-form performance. Style them to match current trends:
CAPTION_STYLES = {     "bold_highlight": {         "description": "Word-by-word animation, keyword in accent color, bold font",         "font": "Montserrat-Black",         "base_color": "#FFFFFF",         "highlight_color": "#FFD700",  # Gold for keywords         "stroke_color": "#000000",         "stroke_width": 3,         "position": "center-bottom",  # 70% down the frame         "animation": "word_by_word",  # Each word appears as spoken         "max_words_visible": 5,       # Rolling window of visible words     },     "minimal_clean": {         "description": "Subtle, lowercase, thin font, no highlight",         "font": "Inter-Regular",         "base_color": "#FFFFFF",         "stroke_color": "#000000",         "stroke_width": 2,         "position": "center-bottom",         "animation": "phrase_by_phrase",  # 3-5 words appear together         "max_words_visible": 8,     },     "tiktok_pop": {         "description": "Colorful, bouncy, emoji accents, all caps keywords",         "font": "Poppins-Bold",         "base_color": "#FFFFFF",         "highlight_color": "#FF3B5C",  # TikTok pink         "stroke_color": "#000000",         "stroke_width": 4,         "position": "center",         "animation": "word_by_word_scale",  # Words pop in with scale animation         "max_words_visible": 4,     } } 
Implementation: Use FFmpeg's drawtext filter with per-word timestamps from Whisper. For the "pop" animation, use the enable flag with timestamp expressions to show/hide words.
For more complex animations (scale, bounce), generate caption frames as transparent PNGs with Pillow and overlay them in FFmpeg.

13. Key Implementation Notes
For Claude Code / the developer:
1. Start with the backend pipeline working end-to-end in a CLI before building the frontend. Get python -m backend.main process_url "https://youtube.com/watch?v=..." working first.
2. The Claude prompts are the most important part. Spend time iterating on the prompts in prompts/. The scoring system prompt should be treated as a living document that improves over time.
3. Use faster-whisper not standard whisper. It's 4-6x faster with the same accuracy. Use the large-v3 model for best results, or medium if the machine can't handle it.
4. For the face tracker, mediapipe's face detection is fast enough to run on every 10th frame. Interpolate positions between detections with an exponential moving average (alpha=0.3) to get smooth panning.
5. Caption burn-in is the hardest FFmpeg part. The word-by-word animation requires generating a complex drawtext filter chain. Consider pre-rendering caption frames as transparent PNGs with Pillow and overlaying them — it's slower but much more controllable.
6. Transcript length management: For long videos (30+ min), the full transcript may exceed Claude's context window. Chunk it into segments with overlap, detect clips per segment, then deduplicate. Alternatively, summarize sections and only send full transcript for the top candidates.
7. Cost optimization: Use claude-sonnet-4-20250514 for all Claude calls (good balance of quality and cost). Cache transcripts and signal analysis so re-runs don't reprocess.
8. Testing: Create a test suite with 3-5 YouTube URLs from different lifestyle/comedy creators. Run the full pipeline and manually evaluate quality. This is how you'll tune the prompts.
